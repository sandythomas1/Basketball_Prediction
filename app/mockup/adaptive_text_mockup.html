Adaptive Text in Donut Chart

5 Different Approaches for Handling Variable-Length Team Names

Approach 1

Dynamic Text Scaling Recommended

Automatically adjust font size based on text length to ensure it always fits within the donut's inner circle.

Pros
  • Always fits perfectly
  • Single-line layout is clean
  • No truncation or information loss
Cons
  • Font size varies between games
  • Requires calculation logic

Approach 2

Truncation with Ellipsis

Use a fixed font size and truncate text with "..." when it exceeds the available width.

Pros
  • Consistent font sizing
  • Simple to implement
  • Prevents overflow
Cons
  • Information is hidden
  • Looks incomplete
  • Poor user experience

Approach 3

Multi-line Text

Break the label into multiple lines: team name on one line, "Win" on another.

Pros
  • All information visible
  • Consistent font size
  • Always readable
Cons
  • Uses more vertical space
  • Can feel cluttered
  • May still wrap awkwardly

Approach 4

External Label Alternative

Move team name outside the donut, keep only percentage and "Win Probability" inside the center.

  • No space constraints
  • Cleaner design
  • More professional look
Cons
  • Less immediate connection
  • Requires more screen space

Approach 5

Abbreviations Only

Use 3-letter NBA team codes instead of full names (e.g., "GSW Win", "LAL Win").

Pros
  • Always fits perfectly
  • Consistent sizing
  • Compact and clean
Cons
  • Less user-friendly
  • Requires NBA knowledge
  • Not intuitive for new users
